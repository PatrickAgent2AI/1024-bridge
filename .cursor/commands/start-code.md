# start-code

我现在需要你和我一起按照TDD（测试驱动开发）的方法，来开发一个项目。现在你需要生成代码。要求如下：

1. 代码逻辑和细节参考 docs/API-SPEC.md 文档，不要遗漏任何功能点。
2. test文件夹下的脚本代码中，一定要将代码块注释取消掉，不要遗漏任何测试用例。
3. 代码实现之后使用 anchor test 命令进行测试，确保测试用例通过。根据测试执行输出分析错误原因。根据错误原因修改实现代码，直到测试用例通过为止。
4. 如果怀疑是测试套件的问题，向我提示，但不要修改测试套件代码。根据我的指令继续代码实现的工作。
5. 只能修改我指定统一修改的测试套件代码，其他任何代码都不要修改。之后再需要修改依然需要向我请示。
6. 必须要通过修改测试套件才能通过测试的用例，必须总结之后向我解释原因。
